# Operations Documents

このディレクトリには、運用時に必要になる手順や判断基準を配置する。

主な対象は、リリース手順、障害対応、環境変数、監視、バックアップ、データ修正手順である。

## MVPの運用方針

- バックエンドサービス、データベース、監視サービスは持たない。
- アプリはテレメトリー、更新確認、クラッシュ送信を行わない。
- 音声、文字起こし結果、運用ログを永続保存しないため、バックアップとデータ復旧の対象はない。
- MVPではコード署名、公証、第三者向けリリース手順を定義しない。
- 配布を開始する際は、署名、公証、ライセンス文、対応macOS、成果物形式をこのディレクトリへ追加する。
