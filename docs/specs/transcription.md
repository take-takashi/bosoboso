# 継続文字起こし仕様

## 起動と終了

1. アプリは `~/.config/bosoboso/config.toml`、辞書、Whisper モデルの順に検証する。
2. モデルの読み込み中は音声を取り込まず、処理中であることを表示する。
3. モデル読み込み後に macOS のマイク権限を要求し、許可後に音声取り込みを開始する。
4. 小窓を閉じた場合は、処理待ち音声と表示内容を保存せずにアプリを終了する。
5. アプリは同時に一つだけ起動できる。二重起動時は既存の小窓を前面へ出す。

## 音声取り込み

- 起動中は、一時停止されるまで選択したマイクの音声を取り込む。
- 一時停止は新しい音声の取り込みだけを停止する。取得済みまたは処理待ちの音声は処理する。
- 再開時は新しい発話区間として取り込みを開始する。
- 初期状態では macOS のデフォルト入力デバイスを使用する。
- 小窓から、現在の起動中に使用する入力デバイスを変更できる。選択は保存しない。
- 使用中のデバイスが切断された場合は、進行中の区間を閉じ、文脈をリセットし、システムのデフォルト入力へ切り替える。
- 入力デバイスが存在しない場合は待機状態とし、デフォルト入力が利用可能になった時点で自動再開する。
- システム音声は直接取り込まない。

## 発話区間と文章ブロック

- Silero VAD によって、内部処理用の発話区間を検出する。
- 発話区間を確定する無音時間の初期候補は 1.2 秒とし、設定で変更可能にする。
- 複数の発話区間は、利用者が扱う一つの文章ブロックへ連結できる。
- 新しい文章ブロックへ切り替える無音時間の初期候補は 10 秒とし、設定で変更可能にする。
- 無音がなく話し続けた場合は、初期候補として 30 秒で内部区間を強制分割する。
- 強制分割時は前後の音声を重ねて認識し、重複した文字列を結合時に除去する。
- 内部区間の境界では強制改行せず、日本語には原則として空白を加えない。英単語間には必要な空白を維持する。
- 最大区間長と音声の重複時間は設定可能にする。

各初期値は技術検証用の値であり、実機評価後に変更できる。

## 暫定結果と確定結果

- 発話中は、編集できない暫定結果を最新位置へ表示する。
- 発話区間の処理後に確定結果へ置き換え、辞書補正を適用する。
- 同じ文章ブロックでは、初期状態で直前の認識結果を Whisper の文脈として引き継ぐ。
- 文脈引き継ぎは設定で無効化できる。
- 新しい文章ブロック、すべて消去、ユーザー編集、マイク切り替え、辞書再読み込みで文脈をリセットする。
- フィラーは削除せず、Whisper が認識した内容を残す。

## 処理待ち

- 推論が音声入力へ追いつかない場合は、音声区間をメモリ上の処理待ちキューへ積む。
- 処理待ち音声の初期上限は 60 秒とし、設定で変更可能にする。
- 上限超過時は古い未処理区間から破棄し、欠落が発生したことを小窓へ明示する。
- 音声データはディスクへ退避しない。

## 文章ブロックの操作

- 最新の文章ブロックを画面上部へ表示する。
- 各文章ブロックには、確定時刻、編集可能な本文、本文コピーボタンを表示する。
- 時刻は画面表示だけに使い、コピー内容には含めない。
- 直近の確定ブロックをコピーする操作を小窓上部へ置く。暫定結果は対象にしない。
- 通常のテキスト選択によって、任意範囲をコピーできる。
- コピー後も文章ブロックを残す。
- ユーザーが編集を開始した文章ブロックは閉じ、新しい認識結果を別のブロックへ追加する。
- 個別削除と Undo は提供しない。
- 「すべて消去」は確認後に表示済みブロックを消し、操作時点以前の処理結果も表示しない。音声取り込みは継続する。

## 履歴表示

- 履歴はメモリ上だけに保持する。
- 保持する文章ブロック数の初期上限は 100 件とし、設定で変更可能にする。
- 上限を超えた場合は、最も古いブロックから破棄する。
- 過去の文章を閲覧、選択、編集している間は表示位置を固定する。
- 表示位置を固定している間に結果が追加された場合は、「新しい結果があります」と表示し、操作によって最新位置へ戻れるようにする。

## 小窓

- 初期状態では、ほかのアプリより手前へ表示する。
- ピン操作によって、通常のウィンドウ表示へ切り替えられる。
- 閉じる操作はアプリ終了として扱う。
- ウィンドウのサイズと位置は保存しない。
- システムテーマへ追従する。
- 状態は色だけに依存せず、「取り込み中」「一時停止」「処理待ち」「入力デバイス待機中」「エラー」などの文字を併記する。
- 波形は表示せず、簡単な入力レベルを表示する。
- 主要操作はキーボードだけでも実行できる。
- UI の表示言語は MVP では日本語だけとする。

## エラー処理

- 設定、辞書、モデルに不備がある場合は、文字起こし機能を利用不可にする。
- エラー画面に原因、可能であればファイル位置、再読み込みまたは再試行、終了を表示する。
- 以前の辞書、暗黙の既定値、部分的に読み込めた項目へフォールバックしない。
- マイク権限が拒否された場合は、macOS の設定方法と再試行操作を表示する。
- 入力デバイスの一時的な不在は回復可能な待機状態として扱う。

## 辞書再読み込み

1. 再読み込み操作時の音声取り込み状態を記録する。
2. 新しい音声の取り込みを止める。
3. 既存の処理待ちを現在の辞書で完了する。
4. 新しい辞書全体を検証して切り替える。
5. 文章ブロックを閉じ、Whisper の文脈をリセットする。
6. 再読み込み前に取り込み中だった場合だけ、取り込みを再開する。

再読み込みに失敗した場合は、文字起こしを利用不可とする。
