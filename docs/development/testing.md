# テスト戦略

## テスト層

| 層 | 対象 | モデル | 実行場所 |
| --- | --- | --- | --- |
| ユニットテスト | 設定検証、相対パス、辞書、区間結合、ブロック化、キュー上限、状態遷移 | 不要 | ローカル、CI |
| GUIコンポーネントテスト | 表示、編集、コピー、一時停止、エラー、キーボード操作 | 偽物 | ローカル、CI |
| Integration Test | 音声ファイルからVAD、Whisper、辞書補正まで | 必要 | PR作成前のローカル |
| E2Eテスト | パッケージ済みアプリへの疑似音声入力から画面表示まで | 必要 | PR作成前のローカル |
| 手動実機テスト | 実マイク、権限、デバイス切り替え、常に手前への表示 | 必要 | Apple Silicon Mac |

## 自動テストの境界

- Integration TestとE2Eテストでは、リポジトリ外の音声ファイルとWhisperモデルを使用する。
- 音声ファイルとモデルはリポジトリへ含めない。
- 対応するmiseタスクは、音声ファイルとモデルのパスを`usage`の必須引数として受け取る。
- 通常CIは外部モデルをダウンロードせず、Integration TestとE2Eテストを実行しない。
- PRテンプレートへ、ローカルで実行したタスク、モデル名、macOSバージョン、Apple Silicon機種、結果を記載する。
- 音声内容と文字起こし結果そのものは、PRへ記載しない。

## GUIコンポーネントテスト

Fyneのテストドライバーを使い、実ウィンドウを表示せずに検証する。音声入力、Whisper、クリップボードは差し替え可能にし、次のフローを対象にする。

1. 暫定結果を受け取る。
2. 編集不可の暫定表示を確認する。
3. 確定結果と辞書補正を受け取る。
4. 編集可能な文章ブロックを確認する。
5. 本文コピーボタンを操作する。
6. テスト用クリップボードの本文を確認する。

過去を閲覧中の表示位置固定、直近コピー、すべて消去、辞書エラー、入力デバイス待機も検証対象とする。

## 品質ゲート

- 一律のカバレッジ最低値は設定しない。
- 重要な振る舞いには、変更と同時にテストを追加する。
- CIでは、フォーマット確認、golangci-lint、ユニットテスト、GUIコンポーネントテスト、モデル不要のビルドを必須とする。
- 実装後のすべての検証入口はmiseタスクとして定義する。

## 手動確認

- 初回起動時にマイク権限が要求される。
- 権限拒否時に設定方法と再試行が表示される。
- 実マイクの音声が暫定表示と確定表示へ反映される。
- 入力デバイスを切り替えられる。
- 使用中のデバイスを切断すると、デフォルト入力へ切り替わる。
- 入力デバイスがない場合は待機し、復帰後に再開する。
- ピン操作によって常に手前への表示を切り替えられる。
- 小窓を閉じるとアプリが終了し、履歴が残らない。
