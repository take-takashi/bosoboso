# ADR 0001: 完全ローカルの単一プロセス構成を採用する

## 状態

採用

## 背景

音声や開発用プロンプトをクラウドへ送信せず、通常のデスクトップアプリとして簡単に起動・終了できる必要がある。また、可能な限り配布しやすくし、Homebrew、Python、FFmpegなどの外部ランタイムを利用者へ要求したくない。

## 決定

- 音声入力、VAD、Whisper推論、辞書補正、GUIを一つのプロセス内で実行する。
- 実行中のネットワーク通信、テレメトリー、更新確認、クラッシュ送信を行わない。
- 音声と文字起こし結果をディスクへ保存しない。
- 運用ログはファイルへ保存せず、標準エラー出力だけに記録する。音声と文字起こし内容はログへ含めない。
- 配布物へ推論ライブラリとVADモデルを組み込み、利用者にはWhisperモデルと設定・辞書だけを外部ファイルとして要求する。

## 結果

- プライバシー方針を説明しやすくなる。
- モデルの推論性能とメモリ使用量は利用者のMacと選択モデルに依存する。
- 実行中の履歴復元、クラッシュ解析の自動送信、自動更新は提供できない。
- 配布時には、cgo依存を含むmacOSアプリのビルドとライセンス表示が必要になる。
