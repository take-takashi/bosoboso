# ADR 0003: Fyneを採用し、OS境界だけを実機テストへ分離する

## 状態

採用

## 背景

Goを中心にクロスプラットフォームの小窓を実装し、GUIの主要な利用フローを自動テストしたい。実マイク、macOSの権限、常に手前への表示などは、ヘッドレスなGUIテストだけでは保証できない。

## 決定

- GUIにはFyneを採用する。
- GUIはアプリケーション状態とイベントだけで動作し、音声入力やWhisperを直接呼ばない。
- Fyneのテストドライバーを使い、表示、編集、コピー、一時停止、キーボード操作、エラー状態をヘッドレスに検証する。
- 音声ファイルを疑似入力として注入するIntegration TestとE2Eテストをローカルで実行する。
- 実マイク、マイク権限、デバイス抜き差し、常に手前への表示は、macOS実機の手動チェックへ分離する。

## 結果

- 通常のGoテストでGUIの主要動作を高速に検証できる。
- プラットフォーム固有処理を小さなアダプターへ隔離する必要がある。
- 完全な自動E2Eテストは目指さず、ローカルの疑似音声E2Eと手動チェックを組み合わせる。
