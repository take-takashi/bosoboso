# ADR 0002: Whisperモデルを外部指定し、辞書補正を先行する

## 状態

採用

## 背景

Whisperモデルは品質によって数百MiBから数GiBになり、利用者が望むモデルも異なる。品質の低いモデルを同梱しても使われず、配布物だけが肥大化する。一方で、`mise`などの固有表記を改善するには、単純な一般音声認識だけでは不足する。

## 決定

- Whisperモデルは同梱せず、設定ファイルのパスで指定する。
- 絶対パスと、設定ファイルの配置ディレクトリを基準とする相対パスを許可する。
- MVPでは`whisper.cpp`が扱えるGGML形式だけに対応する。
- 辞書の正規表記と読み方をWhisperの初期プロンプトへ渡す。
- 利用者が明示した誤認識候補だけを、確定結果へ決定的に置換する。
- 別のローカルLLMによる校正はMVPへ含めない。
- Silero VADモデルは小さく、アプリの内部実装として固定できるため同梱する。

## 結果

- アプリ本体を小さく保ち、利用者が速度と精度の異なるモデルを交換できる。
- 初回利用前に、利用者がモデルを用意して設定する必要がある。
- Whisperは発音辞書を直接扱わないため、「みーず」を必ず`mise`へ変換する保証はない。
- 明示的置換には、本来の「水」を`mise`へ変えるような誤補正の可能性があるため、登録項目だけに限定する。
- LLM校正の必要性は、実際の認識結果を評価した後に再検討する。
